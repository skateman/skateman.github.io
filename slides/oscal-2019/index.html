What do we want?

  • Access to resources

    • High level of automation

Storage

  • In UNIX "everything" is a file
  • Access to devices as they would be files
    • Writing into the file writes to the device
    • Reading from the file reads from the device
  • Mounting
    • Connect a drive to the directory tree
    • Filesystem driver does the hard work
  • Access to files as they would devices
    • Create a file and format it as a drive
    • Mount the file as a drive
  • Files can be shared over the network
# Create 256 MB file filled with zeroes
dd if=/dev/zero of=my-drive bs=1M count=256

# Format the file as FAT32
mkfs.vfat -F 32 my-drive

# Create a folder to mount the file
mkdir drive-content

# Mount the file to the directory tree
mount my-drive drive-content

# Write something into a file
echo "hello world" > drive-content/readme

# Unmount the file from the directory tree
umount drive-content

# Remove the empty folder
rmdir drive-content

Containers

  • Confined environment
    • Very similar to chroot
    • Privileged or unprivileged
  • Disk image file
    • Your stuff + dependencies
    • Base images from container registry
    • Can be built using definition files
  • Virtual network interface
    • External network access
    • Can expose services

Kubernetes

  • Container orchestration platform
  • Run containers on multiple machines
  • Pods
    • Smallest units defining a service
    • Container(s), storage, settings
    • Grouped into services
  • Highly scalable
    • Replication
    • Load balancing
  • You care only about your app
